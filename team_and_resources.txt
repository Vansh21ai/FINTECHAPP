DEV 1  —  THE BACKEND ARCHITECT
----------------------------------------------------------------

  What you build:
  The entire server, database, and the Financial Event Bus.
  Nothing in the app saves or moves without your work.

  Your tasks:

      1.  Node.js + Express API
          Set up all routes:  /user  /invest  /rewards  /trust-log  /insurance

      2.  PostgreSQL Database  (on Neon — free)
          Create tables for:
          - Append-only transaction ledger
          - Consent log (for Trust feature)
          - User XP and tier tracking
          - Rewards and micro-invest tracking

      3.  Financial Event Bus
          Use BullMQ + Upstash Redis.
          4 queues:  invest / reward / insurance / trust
          When a transaction fires → all 4 queues trigger at once.

      4.  Auth
          JWT tokens for login sessions.
          bcrypt for password hashing.
          Rate limiting on all financial routes.

  Free tools:

      Neon (PostgreSQL)     →  https://neon.tech
      Upstash (Redis)       →  https://upstash.com
      Railway (hosting)     →  https://railway.app
      BullMQ docs           →  https://docs.bullmq.io

  npm installs:
      npm install express jsonwebtoken bcryptjs
      npm install bullmq dotenv express-rate-limit

DEV 2  —  THE FRONTEND ENGINEER  +  ANDROID WRAPPER
----------------------------------------------------------------

  What you build:
  Everything the user sees — dashboard, charts, animations, Android app.

  Your tasks:

      1.  React App + Bento Grid Dashboard Layout
          Use Tailwind CSS for the grid.
          Apply glassmorphism to all cards.
          Set mesh gradient as the global background.

      2.  All Charts using ApexCharts
          - Line chart  →  Risk Simulation Sandbox
          - Speedometer gauge  →  Loss Probability Meter
          - Circular bar  →  Trust Score
          - Area curve  →  Compound Interest (Micro-Invest)
          - Progress bar  →  XP in the header

      3.  Onboarding Flow
          3 animated question cards using Framer Motion.
          On submit → POST to backend → get config → load dashboard.

      4.  Lottie Animations
          Confetti  →  savings milestone hit
          Coin spin  →  reward auto-invested
          Badge unlock  →  XP level up

      5.  Android Studio TWA Wrapper  (do this last, after deployment)
          Wraps the deployed web app into a native Android shell.
          Full screen. No browser bar. Looks 100% like a real app.

          Steps:
          - New project in Android Studio → "No Activity" template
          - Add androidbrowserhelper to build.gradle
          - Point manifest to your Cloudflare Pages URL
          - Generate SHA-256 fingerprint
          - Host assetlinks.json on the frontend domain
          - Run on emulator or USB phone

DEPLOYMENT STACK  —  Fast, Free, No Cold Starts
----------------------------------------------------------------

  Frontend   →  Cloudflare Pages   →  https://pages.cloudflare.com
               Serves from 300+ global edge locations.
               No cold starts. Always instant. Completely free.

  Backend    →  Railway            →  https://railway.app
               Always-on Node.js server. Never sleeps. Free tier is enough.

  Database   →  Neon               →  https://neon.tech
               Free serverless PostgreSQL. Wakes in milliseconds.

  Redis      →  Upstash            →  https://upstash.com
               Free serverless Redis. Powers cache + BullMQ queues.


  Why not Vercel?
  Vercel free tier puts functions to sleep after inactivity.
  First user to open the app waits 5–10 seconds for it to wake up.
  That is not acceptable for a live demo. Cloudflare Pages has no sleep.
