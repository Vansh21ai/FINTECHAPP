        FILE 1  —  WHAT WE ARE BUILDING AND WHY
        Gen-Z Financial Super App  |  Open Innovation Hackathon
================================================================

We are combining 4 financial problems into ONE unified web app.
Each section below explains the feature, why we added it, and what it solves.

----------------------------------------------------------------


FEATURE 1  —  RISK SIMULATION SANDBOX
Solves: Investing Fear

  Why we added it:
  Young users want to invest but are terrified of losing money.
  Nobody wants to risk real money without understanding how it works.

  What it does:
  User enters a fake amount (say Rs. 10,000).
  They pick an investment type — Mutual Fund, Stocks, or Gold.
  The app shows what would have happened to that money
  using real historical market data (COVID crash, 2021 boom etc.)
  An AI text box explains what happened in simple words.

  Visuals needed:
  → A line graph going UP and DOWN  (like a stock chart)
  → A speedometer gauge  —  Green = Safe,  Red = Risky


----------------------------------------------------------------


FEATURE 2  —  CONSENT LEDGER
Solves: Trust Deficit

  Why we added it:
  Gen-Z does not trust apps. They think their data is being sold.
  No competitor shows users what happens to their data in real time.
  This feature makes us different from every other fintech app.

  What it does:
  Every time the app reads user data, one log line appears:

      10:32 AM  →  Location checked     →  Fraud prevention
      10:33 AM  →  Spending data read   →  Building your investment tip
      10:35 AM  →  Phone model checked  →  Checking insurance eligibility

  User can open this log any time and see exactly what happened.
  Also — if a payment fails, the app gives a human reason:
  "Payment stopped — this merchant was flagged 5 times today."

  Visuals needed:
  → A live scrolling feed  (Green = safe,  Yellow = advisory,  Red = sensitive)
  → A circular Trust Score bar  (0 to 100)


----------------------------------------------------------------


FEATURE 3  —  AUTO MICRO-INVEST  +  FINANCIAL XP SYSTEM
Solves: Cashback Dependency

  Why we added it:
  Other apps give Rs. 5 cashback to make users spend more.
  That builds zero wealth. We flip this into a savings habit instead.
  We added the XP System to make saving feel like a game — not a chore.

  What it does:
  When a user buys coffee for Rs. 120, they earn a reward.
  Instead of cashback to wallet, that reward is AUTO invested
  into a liquid mutual fund. Over time it grows on the dashboard.

  XP System — users earn points for good financial actions:

      Complete KYC              →  +100 XP
      First micro-invest        →  +200 XP
      Saved 3 months in a row   →  +500 XP
      Added insurance cover     →  +300 XP

  Tiers:  Beginner → Saver → Investor → Wealth Builder → Money Master

  When user hits "Investor" tier, the Risk Sandbox unlocks with live data.
  Milestones trigger confetti animation and a badge unlock.

  Visuals needed:
  → A compound interest curve  (shows how small savings grow big over time)
  → An XP progress bar in the top header  (always visible)


----------------------------------------------------------------


FEATURE 4  —  INSURANCE SCENARIO SIMULATOR
Solves: Insurance Complexity

  Why we added it:
  Insurance policies are written in legal language nobody reads.
  Youth skip insurance and only regret it after something goes wrong.

  What it does:
  User sees 4 scenario buttons:

      "What if I drop my phone in a pool?"
      "What if my laptop gets stolen?"
      "What if I get in a bike accident?"
      "What if a cyber fraud empties my account?"

  They click one. The app instantly responds:

      COVERED ✅  —  "Your Gadget Plan covers accidental water damage."
      NOT COVERED ❌  —  "Your plan doesn't cover this. Upgrade for Rs. 29/month."

  Visuals needed:
  → A 3D umbrella covering phone, laptop, wallet icons
  → Icons under umbrella  =  Green (covered)
  → Icons outside umbrella  =  Red (not covered)
  → Toggle switches let users add coverage and watch icons turn green live


----------------------------------------------------------------


THE FINANCIAL EVENT BUS  —  What Connects Everything
Why we added it:

  Without this, we have 4 separate features. Not a super app.
  This is the hidden engine that makes all 4 modules work together.

  When a user buys a phone (Rs. 45,000) — all 4 modules react at once:

      Insurance   →  Suggests gadget cover automatically
      Cashback    →  Auto-invests the reward
      Trust       →  Logs the transaction in the consent ledger
      Investment  →  Suggests phone-sector stocks as a tip

  Technically this is a simple event system in Node.js.
  It is how real fintech companies like Razorpay work internally.
  Our hackathon version is a genuine scaled-down implementation.


----------------------------------------------------------------


THE 3-QUESTION ONBOARDING
Why we added it:

  Without onboarding, every user sees the same dashboard.
  Gen-Z abandons any app that feels generic in the first 30 seconds.
  3 questions personalizes the entire dashboard before they even see it.

  The 3 questions:

      1.  What is your monthly income?
          Below Rs. 15k  /  Rs. 15k–40k  /  Rs. 40k+

      2.  What is your biggest financial worry?
          Losing money  /  Not saving  /  Don't understand investing

      3.  Pick your first goal
          Emergency fund  /  Travel  /  Start a business

  Based on answers — the dashboard shows the most relevant feature first.
  User scared of losses → Trust Dashboard is front and center.
  User wants to save → Micro-Invest tracker is the hero panel.
