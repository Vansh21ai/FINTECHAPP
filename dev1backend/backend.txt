================================================================================
          BACKEND ARCHITECT (DEV 1) - STEP-BY-STEP STARTING GUIDE
================================================================================

Welcome, Architect! Your job is to build the "Brain" of the Gen-Z Fintech App.
Without your code, nothing moves, nothing saves, and nothing is secure.

This guide will take you from a blank folder to a running financial engine.

--------------------------------------------------------------------------------
1. PROJECT INITIALIZATION (The Foundation)
--------------------------------------------------------------------------------

   Step 1: Open your terminal in the `dev1backend` folder.
   Step 2: Run the following command to create your package.json:
           
           npm init -y

   Step 3: Create your folder structure to stay organized:
           
           mkdir src
           mkdir src\routes
           mkdir src\controllers
           mkdir src\models
           mkdir src\config
           mkdir src\queues
           touch src\index.js
           touch .env

--------------------------------------------------------------------------------
2. DOWNLOADING ALL PACKAGES (The Tools)
--------------------------------------------------------------------------------

   Run these two commands to install everything you need:

   Command A (Core Web Server & Security):
   npm install express jsonwebtoken bcryptjs dotenv cors helmet express-rate-limit

   Command B (Database & Event Bus):
   npm install pg bullmq ioredis

   Command C (Development Tools - helps you restart server automatically):
   npm install --save-dev nodemon

--------------------------------------------------------------------------------
3. SETTING UP YOUR FREE CLOUD SERVICES (The Engines)
--------------------------------------------------------------------------------

   You need two main "engines" running in the cloud (they are free):

   A. NEON POSTGRESQL (Your Database)
      1. Go to https://neon.tech and create a free account.
      2. Create a new project called "FintechApp".
      3. Copy the "Connection String" (it looks like postgres://user:password@host/db).
      4. Paste it into your `.env` file like this:
         DATABASE_URL=your_connection_string_here

   B. UPSTASH REDIS (Your Event Bus Engine)
      1. Go to https://upstash.com and create a free account.
      2. Create a "Redis" database.
      3. Scroll down to the "Node.js" section and copy the credentials.
      4. Paste them into your `.env` file:
         REDIS_URL=your_redis_url_here

--------------------------------------------------------------------------------
4. THE TASK BREAKDOWN (Pointer by Pointer)
--------------------------------------------------------------------------------

   Task 1: Setup Express Server (`src/index.js`)
           - Initialize express, cors, and helmet.
           - Connect to PostgreSQL using the `pg` package.
           - Start server on Port 3000.

   Task 2: Setup Database Schema
           - Create the following tables in Neon:
             - `users`: (id, email, password_hash, xp, tier)
             - `ledger`: (id, user_id, amount, type, description, timestamp)
             - `consent_logs`: (id, user_id, action, reason, timestamp)
             - `rewards`: (id, user_id, amount, invested_at)

   Task 3: Implement JWT Authentication
           - Create `/auth/register` (hash password with bcrypt).
           - Create `/auth/login` (return a JWT token).

   Task 4: Build the Financial Event Bus (The Secret Sauce)
           - Setup BullMQ to use your Upstash Redis.
           - Create 4 queues: `invest`, `reward`, `insurance`, `trust`.
           - Create a "Producer" function that fires when a transaction happens.
           - Create "Consumers" that listen to these queues and update the DB.

   Task 5: Create Financial Routes
           - `POST /transaction`: This is where the magic happens.
           - `GET /user/xp`: Fetch current XP and tier.
           - `GET /trust-log`: Fetch the consent ledger history.
           - `GET /invest/simulation`: Historical growth data for the sandbox.

--------------------------------------------------------------------------------
5. HOW TO USE AI TO GENERATE THE CODE
--------------------------------------------------------------------------------

   Don't write everything from scratch. Use your AI (like me or a LLM) with these
   specific prompts. Copy-paste them to get high-quality code:

   Prompt 1 (For Server Setup):
   "Generate a Node.js Express server index.js file using 'dotenv' for config. 
   Include 'helmet' and 'cors' for security. Add a basic health check route 
   at '/' and connect to a PostgreSQL database using the 'pg' library."

   Prompt 2 (For Event Bus):
   "Show me how to set up BullMQ for a Node.js app using Upstash Redis. 
   I need 4 queues: invest-queue, reward-queue, insurance-queue, and trust-queue. 
   Write a function that accepts a transaction and adds tasks to all 4 queues."

   Prompt 3 (For Database Schema):
   "Write the SQL 'CREATE TABLE' statements for a fintech app ledger system. 
   I need a users table, an append-only transaction ledger, a consent_logs table 
   for privacy tracking, and an XP tracking system for Gen-Z users."

--------------------------------------------------------------------------------
6. RUNNING AND TESTING YOUR WORK
--------------------------------------------------------------------------------

   Update your `package.json` with this script:
   "scripts": {
     "start": "node src/index.js",
     "dev": "nodemon src/index.js"
   }

   To start your server for development:
   
   npm run dev

   To test your API:
   - Use Postman or Thunder Client (VS Code Extension).
   - First, try `GET http://localhost:3000/` to see if the server is alive.

--------------------------------------------------------------------------------
7. FINAL CHECKLIST (Before you merge on Day 2)
--------------------------------------------------------------------------------

   [ ] Does registration return a JWT?
   [ ] Does a transaction log appear in the PostgreSQL ledger?
   [ ] Does the Event Bus trigger all 4 queues correctly?
   [ ] Is the `.env` file listed in your `.gitignore`? (CRITICAL)
   [ ] Can the frontend reach your API without CORS errors?
